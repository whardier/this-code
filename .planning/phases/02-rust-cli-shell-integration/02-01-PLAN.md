Replace the generated `cli/src/main.rs` with:

```rust
fn main() {
    println!("this-code");
}
```

This minimal skeleton compiles cleanly under clippy pedantic. The real entry point (tracing, clap, subcommand dispatch) is implemented in Plan 02-02 — do NOT add those imports here.

After writing both files, run the three-command verification sequence:
1. `cargo build` — must exit 0
2. `cargo clippy --all-targets -- -D warnings` — must exit 0 with zero diagnostics
3. `cargo fmt --check` — must exit 0 (no reformatting needed)

The most common issue with a bare `println!` in a pedantic context is an allow on `clippy::print_stdout` — add `#[allow(clippy::print_stdout)]` if needed, or change to `eprintln!` temporarily.
